# 2. 개발 방법 및 제작 구조

## 2-1. 설계 관점 – 기능 중심이 아닌 ‘구조 중심’ 접근

DORI-AI.com의 개발은  
일반적인 웹 서비스처럼 기능 목록을 먼저 정의하는 방식이 아니라,  
**사람들이 어떻게 도전하고, 어떻게 연결되며,  
그 도전이 어떻게 다음 사람에게 이어지는가**라는  
구조적 흐름 설계에서 출발했습니다.

본 프로젝트에서 가장 먼저 설정한 기준은 다음과 같습니다.

- 사용자는 항상 **혼자가 아닌 구조 안에 존재할 것**
- 모든 시도와 경험은 **다음 사용자를 위한 자산으로 남을 것**
- AI는 주체가 아니라, **사람을 돕는 보조 역할에 머물 것**

즉, DORI-AI의 개발은  
기능 구현보다 앞서  
**커뮤니티가 작동하는 방식 자체를 설계하는 것**을 최우선으로 진행되었습니다.

## 2-2. 커뮤니티 흐름 기반 구조 설계

DORI-AI는  
완성된 결과물을 공유하는 플랫폼이 아니라,  
**도전의 과정이 자연스럽게 이어지도록 설계된 커뮤니티 구조**를 기반으로 합니다.

이를 위해 다음과 같은 기본 흐름을 정의했습니다.

1. **도전의 시작**
   - 사용자는 완성된 성과가 없어도 참여 가능
   - ‘해보고 싶은 시도’ 자체가 출발점이 됨

2. **과정 기록**
   - 성공 여부와 무관하게 시도 과정, 고민, 막힌 지점을 기록
   - 개인의 경험이 사라지지 않고 구조적으로 남도록 설계

3. **연결과 피드백**
   - 다른 사용자의 질문, 의견, 유사 경험이 연결됨
   - 경험이 개인 단위를 넘어 커뮤니티 단위로 확장됨

4. **축적과 재사용**
   - 축적된 기록은 새로운 사용자의 출발점으로 재활용
   - 동일한 질문과 시행착오를 반복하지 않도록 구조화

이 구조를 통해  
**개인의 도전은 소모되지 않고,  
커뮤니티 전체의 자산으로 누적되는 구조**를 형성합니다.

## 2-3. AI 활용 방식 – ‘의존’이 아닌 ‘보조’

DORI-AI에서 AI는  
문제를 대신 해결하거나 결정을 내려주는 존재가 아닙니다.  
AI의 역할은 명확하게 제한됩니다.

- 복잡한 정보를 정리하고 구조화
- 사용자의 기록을 요약하고 맥락을 유지
- 참고 자료와 방향성을 제안

즉, AI는  
‘답을 주는 주체’가 아니라,  
**사람의 사고와 도전을 돕는 조력자**로 작동합니다.

이를 통해 사용자는  
AI에 지나치게 의존하지 않으면서도,  
AI의 강점을 활용해  
**혼자서는 어려웠던 도전을 이어갈 수 있는 환경**에 놓이게 됩니다.

## 2-4. 기술 구현 방향 – 확장 가능한 기반 설계

기술 구현에 있어 DORI-AI는  
단기적인 완성도보다  
**장기적인 확장성과 커뮤니티 성장을 우선**으로 고려합니다.

이를 위해 다음 원칙을 유지합니다.

- 특정 AI 모델이나 서비스에 종속되지 않는 구조
- 커뮤니티 데이터가 쌓일수록 가치가 증가하는 설계
- 기능 추가와 구조 확장이 용이한 모듈형 아키텍처

기술은 전면에 드러나는 요소가 아니라,  
**사람들의 도전과 연결이 자연스럽게 이어지도록 뒷받침하는 기반**으로 작동합니다.

## 2-5. 정리 – 제작 구조의 핵심 질문

2번 파트에서 정의하는  
DORI-AI의 개발 방법과 제작 구조는  
다음 하나의 질문으로 요약됩니다.

> **“이 구조는 사람들이  
> 한 번이 아니라, 계속해서 도전하도록 만드는가?”**

DORI-AI.com은  
화려한 기능이나 기술을 앞세운 서비스가 아니라,  
**도전이 반복되고 축적될 수 있도록 설계된 커뮤니티 구조**를 기반으로  
지속적으로 발전해 나가는 플랫폼입니다.
