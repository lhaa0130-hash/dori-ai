## 왜 프롬프트 템플릿인가

AI를 쓰는 사람과 잘 쓰는 사람의 차이는 대부분 **프롬프트 품질**에서 납니다. "요약해줘"와 "다음 글을 핵심 3가지로 요약하되, 각 항목을 2줄 이내로 쓰고, 전문 지식이 없는 30대 직장인이 이해할 수 있는 언어로 작성해줘"는 결과물이 완전히 다릅니다.

이 큐레이션은 **지금 바로 복붙해서 쓸 수 있는 프롬프트 템플릿 30가지**를 엄선했습니다. `[대괄호]` 안의 내용만 바꿔서 쓰세요.

## 📝 글쓰기 & 문서 작성 (1~8)

**① 보고서 초안 작성**
```
[주제]에 대한 업무 보고서를 작성해줘.
- 대상 독자: [직급/부서]
- 분량: A4 1장 분량
- 포함 항목: 현황, 문제점, 원인 분석, 해결 방안, 기대 효과
- 톤: 간결하고 전문적으로
```

**② 이메일 초안 (정중한 거절)**
```
다음 상황에 맞는 정중한 거절 이메일을 써줘.
상황: [상황 설명]
보내는 사람: [나의 역할]
받는 사람: [상대방 역할]
거절 이유: [이유]
대안 제시 여부: [있음/없음]
```

**③ SNS 게시물 작성**
```
[플랫폼: 인스타그램/링크드인/트위터]에 올릴 게시물을 써줘.
주제: [내용]
톤: [캐주얼/전문적/유머러스]
해시태그: [5개 추천]
길이: [플랫폼 최적 길이]
```

**④ 회의록 정리**
```
아래 회의 내용을 공식 회의록 형식으로 정리해줘.
- 날짜·참석자 포함
- 결정사항과 미결사항 구분
- 담당자와 마감일이 있으면 명시
- 다음 회의 안건 제안

[회의 내용 붙여넣기]
```

**⑤ 제안서 구조 잡기**
```
[서비스/제품명]을 [대상 고객]에게 제안하는 제안서 목차를 잡아줘.
제안 목적: [목적]
차별점: [주요 강점 3가지]
예산 범위: [금액]
제안서 분량: [페이지 수]
```

**⑥ 블로그 글 작성**
```
다음 키워드로 검색 최적화된 블로그 글을 써줘.
키워드: [키워드]
타겟 독자: [독자 설명]
글의 목적: [정보 제공/설득/경험 공유]
분량: [약 1,500자]
구조: 도입-본문(소제목 3개)-결론 형식
```

**⑦ 보도자료 작성**
```
[회사명]의 [발표 내용]에 대한 보도자료를 작성해줘.
발표일: [날짜]
주요 내용: [핵심 사항]
인용구: [대표자 이름과 직함]의 코멘트 포함
배포 대상: [언론사 종류]
```

**⑧ 약관·정책 문서 요약**
```
다음 약관/개인정보처리방침을 일반인이 이해하기 쉽게 요약해줘.
- 가장 중요한 5가지 항목 먼저
- 사용자에게 불리할 수 있는 조항 별도 강조
- 전문용어는 괄호 안에 쉬운 말로 설명

[문서 내용 붙여넣기]
```

## 🔍 정보 수집 & 분석 (9~15)

**⑨ 경쟁사 분석**
```
[내 서비스/제품]의 경쟁사인 [경쟁사 이름들]을 분석해줘.
분석 항목: 가격, 주요 기능, 타겟 고객, 강점, 약점
형식: 비교표로 정리
마지막에 시사점 3가지 추가
```

**⑩ 시장 조사 프레임 설계**
```
[산업/시장명] 시장 조사를 위한 프레임워크를 설계해줘.
목적: [의사결정 목적]
수집해야 할 데이터: 1차 데이터 vs 2차 데이터 구분
조사 방법론 추천 (설문/인터뷰/데스크리서치 등)
예산: [예산 범위]
일정: [기간]
```

**⑪ 복잡한 개념 쉽게 설명**
```
[개념/기술/용어]를 설명해줘.
- 대상: [전혀 모르는 비전공자]
- 일상의 비유나 예시를 반드시 사용
- 최대 300자 이내로 핵심만
- 마지막에 한 줄 요약
```

**⑫ SWOT 분석**
```
[회사명 또는 아이디어]에 대한 SWOT 분석을 해줘.
- 각 항목별 5가지씩
- 구체적인 근거 포함
- 분석 후 전략 방향 제안 (SO/ST/WO/WT 전략)
```

**⑬ 데이터 해석**
```
다음 데이터를 분석하고 인사이트를 도출해줘.
분석 목적: [목적]
주목해야 할 패턴: 이상치, 트렌드, 상관관계
결론: 의사결정에 활용할 수 있는 인사이트 3가지

[데이터 붙여넣기]
```

**⑭ 논문/보고서 요약**
```
다음 논문/보고서의 핵심을 요약해줘.
- 연구 목적과 방법론
- 주요 발견 3가지
- 한계점
- 실무 적용 가능한 시사점
- 관련 추가 읽을거리 추천

[내용 붙여넣기]
```

**⑮ 의사결정 분석**
```
[상황 설명]에서 [선택지 A] vs [선택지 B]를 결정해야 해.
- 각 선택지의 장단점
- 리스크와 기회
- 추천 선택지와 그 이유
- 추천 선택 시 주의할 점
```

## 💡 아이디어 발상 & 기획 (16~22)

**⑯ 브레인스토밍 진행**
```
[주제]에 대해 브레인스토밍을 도와줘.
제약 조건: [예산/시간/인력 등]
아이디어 수: 최소 20개
기준: 실현 가능성보다 창의성 우선
마지막에 상위 5개를 선별하고 이유 설명
```

**⑰ 마케팅 캠페인 아이디어**
```
[제품/서비스]의 [타겟층]을 대상으로 하는 마케팅 캠페인 아이디어를 제안해줘.
예산: [금액]
기간: [기간]
목표: [인지도/전환율/리텐션 등]
채널: [온라인/오프라인/소셜미디어]
성공 지표(KPI) 포함
```

**⑱ 강의/발표 구조 설계**
```
[주제]로 [시간]짜리 발표/강의 구조를 설계해줘.
청중: [청중 설명]
목표: 발표 후 청중이 [무엇을] 할 수 있어야 함
포함 요소: 도입 훅, 본론(소제목), Q&A, 행동 촉구
시간 배분 포함
```

**⑲ 제품/서비스 네이밍**
```
[제품/서비스 설명]에 맞는 이름 10개를 제안해줘.
타겟: [고객 설명]
브랜드 톤: [모던/친근/신뢰감/도전적]
조건: 한국어/영어/혼합, 상표 등록 가능성 고려
각 이름에 네이밍 이유 한 줄씩
```

**⑳ 콘텐츠 달력 만들기**
```
[채널: 블로그/인스타/유튜브]의 [기간: 1개월] 콘텐츠 달력을 만들어줘.
주제: [전반적인 콘텐츠 방향]
빈도: 주 [횟수]회 게시
각 콘텐츠별 제목, 핵심 메시지, 형식(글/영상/카드뉴스) 포함
```

**㉑ 문제 해결 프레임**
```
[문제 상황]을 해결하기 위한 단계별 계획을 세워줘.
현재 상황: [현황]
제약 조건: [리소스, 시간, 권한 등]
단기(1개월) / 중기(3개월) / 장기(6개월) 목표 구분
각 단계별 구체적 실행 항목
```

**㉒ 가설 검증 설계**
```
[가설 내용]을 검증하기 위한 실험을 설계해줘.
- 검증 방법론
- 필요한 데이터와 수집 방법
- 성공/실패 판단 기준
- 예상 기간과 리소스
- 예상되는 한계점
```

## 🎯 개인 생산성 & 학습 (23~30)

**㉓ 학습 계획 설계**
```
[목표 스킬/자격증]을 [기간] 안에 달성하기 위한 학습 계획을 만들어줘.
현재 수준: [초보/중급]
하루 투자 가능 시간: [시간]
선호 학습 방식: [영상/책/실습]
체크포인트와 중간 목표 포함
```

**㉔ 피드백 요청**
```
다음 [글/코드/기획안]에 대해 피드백을 줘.
평가 기준: [품질/명확성/실용성/설득력]
내가 특히 걱정하는 부분: [구체적 부분]
개선 우선순위를 1~5위로 제안
긍정적인 부분도 함께 언급

[내용 붙여넣기]
```

**㉕ 자기소개서/이력서 개선**
```
다음 자기소개서를 개선해줘.
지원 직무: [직무]
회사: [회사명]
강조하고 싶은 경험: [경험 2~3가지]
개선 방향: 더 임팩트 있고 구체적으로
길이: [자 수 제한]

[현재 자기소개서 붙여넣기]
```

**㉖ 협상 준비**
```
[협상 상황]을 위한 준비를 도와줘.
내 목표: [원하는 결과]
상대방 예상 관심사: [상대방 입장]
협상 레버리지: [내가 가진 강점]
배트나(BATNA): [합의 실패 시 대안]
협상 스크립트 예시 포함
```

**㉗ 어려운 대화 준비**
```
[어려운 대화 상황]을 위한 대화 스크립트를 작성해줘.
대화 목적: [무엇을 달성하려 하는가]
관계: [상대방과의 관계]
원하는 톤: [부드럽게/직접적으로/공감하며]
예상되는 반발 대응 방법 포함
```

**㉘ 영어 글 교정**
```
다음 영어 글을 교정해줘.
목적: [이메일/에세이/프레젠테이션]
독자: [영어 원어민/비원어민]
수준: [격식체/비격식체]
교정 후 변경한 이유를 간략히 설명

[영어 글 붙여넣기]
```

**㉙ 롤플레이 연습**
```
나는 [역할]이고 너는 [상대방 역할]이야.
상황: [구체적 상황]
목표: [내가 연습하고 싶은 것]
피드백: 대화가 끝나면 잘한 점과 개선점을 알려줘
시작해줘.
```

**㉚ 하루 계획 최적화**

에너지 패턴: [오전 집중형/저녁 집중형]
마감 시간이 있는 항목: [명시]
예상 소요 시간이 있으면 포함

[할 일 목록 붙여넣기]

## 마무리: 프롬프트는 근육이다

좋은 프롬프트는 한 번에 완성되지 않습니다. 처음에는 이 템플릿을 그대로 쓰고, 결과물을 보며 조금씩 수정해보세요. 나만의 언어로, 나의 업무에 맞게 다듬어가는 과정에서 당신만의 프롬프트 라이브러리가 완성됩니다.

AI는 결국 어떤 질문을 하느냐에 따라 달라집니다. 더 좋은 질문이, 더 좋은 AI를 만듭니다. 🍡
